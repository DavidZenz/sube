**Section 4: Bring Your Own Data (FMT-03, per D-06 and D-07)**

Heading: `## Bring Your Own Data`

Write a brief intro paragraph: this section walks through reshaping arbitrary supply-use data into the canonical format.

Sub-heading: `### SUT table preparation`

Numbered checklist (per D-06):
1. Identify which columns in your source data correspond to REP, PAR, CPA, YEAR, and TYPE.
2. Rename or map columns to the canonical names, or supply them with any casing — `import_suts()` will uppercase all column names on import.
3. Choose a layout: long format (all 7 columns including VAR and VALUE) or wide format (5 identity columns with industry codes as column headers). `import_suts()` accepts both.
4. If using wide format: be aware that aggregate demand columns (e.g. total final use) will be melted as industry columns unless they match known WIOD aggregates. Remove or rename them before import if needed.
5. Call `import_suts()` on the file path. Inspect the result with `head()` and `str()` to verify all 7 columns are present and values look correct.

Sub-heading: `### Satellite vector preparation`

Numbered checklist (per D-07):
1. Obtain gross output (GO) by industry from national accounts or your preferred source. This column is required.
2. Optionally gather value added (VA), employment (EMP), and CO2 emissions (CO2) by industry.
3. Structure the data with columns YEAR, REP, and one industry identifier column (any of: IND, INDUSTRY, INDUSTRIES, or INDAGG).
4. Ensure YEAR and REP values match those in your SUT data exactly.
5. Provide one row per (YEAR, REP, INDUSTRY) combination. Pass the resulting data frame to the `inputs` argument of `compute_sube()`.
