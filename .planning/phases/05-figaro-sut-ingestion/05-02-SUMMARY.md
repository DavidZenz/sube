## What Was Built

### `R/import.R` — `read_figaro(path, year, final_demand_vars)`

4-step data.table pipeline implementing D-03, D-06, D-08, D-11, D-12, D-19,
D-20, D-21, D-22:

1. **Directory pairing** (D-11/D-12): lists CSV files, matches supply via
   `[-_]supply[-_]` and use via `[-_]use[-_]` (case-insensitive). Zero or
   multiple matches on either side is a hard error.

2. **Year validation** (D-08): requires 4-digit integer in [1900, 2100].

3. **Per-file processing** (`process_one()`):
   - `fread()` → validate required columns (`refArea`, `rowPi`,
     `counterpartArea`, `colPi`, `obsValue`)
   - Filter primary inputs: `dt[startsWith(rowPi, "CPA_")]` (D-19)
   - Rename + strip: `refArea→REP`, `counterpartArea→PAR`,
     `substring(rowPi, 5)→CPA`, `colPi→VAR`, `obsValue→VALUE` (D-03/D-06)
   - USE-only FD aggregation: sum `final_demand_vars` per `(REP, PAR, CPA)`
     into synthetic `VAR = "FU_bas"` rows (D-20). Validates all codes exist
     in `colPi` before aggregation.

4. **Bind + class tag**: `rbindlist(sup, use)` →
   `.sube_required_columns()` assertion → `class<-c("sube_suts", ...)`
