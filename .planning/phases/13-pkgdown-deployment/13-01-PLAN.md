```yaml
name: pkgdown

on:
  push:
    branches: [main, master]
  workflow_dispatch:

# Required by actions/deploy-pages (D-04)
permissions:
  contents: read
  pages: write
  id-token: write

# GitHub's recommendation for Pages: serialize, do not cancel a live deploy (L-04)
concurrency:
  group: pages
  cancel-in-progress: false

jobs:
  deploy:
    runs-on: ubuntu-latest
    timeout-minutes: 20
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - uses: actions/checkout@v4

      - uses: r-lib/actions/setup-pandoc@v2

      - uses: r-lib/actions/setup-r@v2
        with:
          use-public-rspm: true

      - uses: r-lib/actions/setup-r-dependencies@v2
        with:
          extra-packages: |
            any::pkgdown
            local::.
          needs: website

      - name: Build site
        run: pkgdown::build_site_github_pages(new_process = FALSE, install = FALSE)
        shell: Rscript {0}

      - uses: actions/configure-pages@v6

      - uses: actions/upload-pages-artifact@v5
        with:
          path: docs

      - id: deployment
        uses: actions/deploy-pages@v5
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions runner → GitHub Pages service | OIDC token exchange across process boundary; runner signs a deploy request with its ephemeral identity |
| Pull request source → CI runner | PR author may introduce malicious vignette/Rmd content that executes on the runner during `build_site()` |
| Workflow YAML → GitHub API | The workflow grants itself the minimum permissions it needs; any over-grant widens the blast radius |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-01 | Elevation of Privilege | `pkgdown.yaml` permissions block | mitigate | Pin permissions to exactly `contents: read`, `pages: write`, `id-token: write` per D-04; remove legacy `contents: write`. Grep-verified in acceptance criteria. |
| T-13-02 | Tampering | PR smoke-build workflow (runs attacker-controlled Rmd code) | mitigate | `pkgdown-check.yaml` has `contents: read` only and no `pages:`/`id-token:` — a compromised PR build cannot deploy to Pages or push to the repo. Grep-verified. |
| T-13-03 | Spoofing | `actions/deploy-pages@v5` OIDC handshake | mitigate | `id-token: write` is scoped only to the deploy workflow; verified via grep on permissions block. OIDC issuer-audience binding is GitHub-managed — no additional config needed. |
| T-13-04 | Denial of Service | Pages deploy concurrency | mitigate | `concurrency: { group: pages, cancel-in-progress: false }` per L-04 serializes deploys and protects an in-flight production deploy from superseding pushes. |
| T-13-05 | Tampering | Action version pinning (supply-chain) | accept | Actions pinned to major tags (`@v4`, `@v5`, `@v6`) per 13-RESEARCH.md §Pinned action versions; SHA-pinning not adopted this phase (deferred — discretionary). Risk: a malicious tag reassignment by a compromised maintainer. Rationale for accept: all pinned actions are GitHub-owned (`actions/*`) or r-lib-owned, both high-trust; SHA pinning can be added in a later hardening phase if needed. |
| T-13-06 | Information Disclosure | Workflow logs | accept | No secrets are passed to the build step; `GITHUB_TOKEN` is default-scoped; no custom secrets referenced. Standard GHA logging is sufficient. |
</threat_model>
