### Navigation steps for D-14 (from 13-RESEARCH.md §GitHub Pages Prerequisites & Deploy Mechanics §Manual prerequisite)

1. Navigate to **Settings → Pages**.
2. Under **Build and deployment → Source**, select **GitHub Actions** (not "Deploy from a branch").

### Research rationale for why D-14 is manual (not automated)

- `actions/configure-pages@v6` supports `enablement: true` but requires a PAT or GitHub App with `administration:write` — `GITHUB_TOKEN` at default scope does not suffice.
- `PUT /repos/{owner}/{repo}/pages` with `{"build_type": "workflow"}` — same admin-token constraint.
- Given the one-time nature and the elevated-token cost, manual is the pragmatic call.

### Expected failure mode if D-14 skipped

- `actions/configure-pages@v6` fails fast with "Get Pages site failed. Please verify that the repository has Pages enabled and configured to build using GitHub Actions." (Exact text varies by action version.)
- No partial deploy, no artifact lingers.

**Locked decision (from 13-CONTEXT.md):** User manually enables GitHub Pages Source → "GitHub Actions" in repo settings before first deploy.
