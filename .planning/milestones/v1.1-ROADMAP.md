# Milestone v1.1: Replication, FIGARO & Convenience

**Status:** SHIPPED 2026-04-16
**Phases:** 5-6
**Total Plans:** 7

## Overview

This milestone proved the package reproduces the published paper exactly and added FIGARO SUT ingestion as a validated second data source. Phase 5 delivered `read_figaro()` with a tested synthetic fixture, extended `.coerce_map()` to recognize NACE/NACE_R2 column names, and shipped documentation + pkgdown updates. Phase 6 delivered a gated replication test suite, exported `filter_paper_outliers()` as a first-class outlier helper, and shipped a 9-section narrated vignette that walks researchers from WIOD import through elasticity comparison.

The "Convenience" scope (one-call pipeline, batch helpers) was deliberately deferred: no CNV-requirements were defined, and shipped convenience-shaped work (exporting `filter_paper_outliers`, the replication vignette, pkgdown reshuffle, NEWS entries) landed under existing FIG-/REP- IDs. Pipeline/batch helpers carry forward to v1.2.

**Depends on**: Phase 4 (v1.0 package foundation)
**Requirements**: FIG-01, FIG-02, FIG-03, FIG-04
**Plans**: 4 plans

Plans:
- [x] 05-01: Synthetic FIGARO SUT fixtures and failing test skeletons
- [x] 05-02: Implement `read_figaro()` in R/import.R with export + man page
- [x] 05-03: Extend `.coerce_map()` with NACE/NACE_R2 synonyms
- [x] 05-04: Release plumbing — pkgdown group, NEWS v1.1 entry, rowPi globalVariable

**Details:**
Phase 5 turned FIGARO industry-by-industry SUTs into a first-class input: `read_figaro()` parses composite country-industry column labels into REP/PAR/CPA/VAR fields, strips `CPA_` prefixes, aggregates final-demand columns into `FU_bas`, and tags the output with the `sube_suts` class. `.coerce_map()` gained NACE/NACE_R2 synonyms so FIGARO-shaped mapping tables round-trip through `build_matrices()` without falling back to positional matching. Synthetic supply/use fixtures ship in `inst/extdata/figaro-sample/` and drive 46 green testthat expectations. Tarball `R CMD check` exits Status: OK.

## Milestone Summary

**Key Accomplishments:**
- Shipped `read_figaro()` for FIGARO industry-by-industry SUT CSVs with 46 green testthat expectations and synthetic fixtures.
- Extended `.coerce_map()` to recognize NACE and NACE_R2 column names, closing the positional-fallback gap for FIGARO-shaped inputs.
- Delivered a gated replication test suite (REP-01) that numerically reproduces paper results at `1e-6` tolerance with clean CRAN/CI skip behavior.
- Promoted `filter_paper_outliers()` to a first-class export with typed `variables`/`apply_bounds` arguments and full roxygen docs.
- Published a 9-section paper-replication vignette (REP-02) covering WIOD import through elasticity comparison with `eval=FALSE`.
- Reorganized the pkgdown reference into a `Paper replication tools` group and logged three v1.1 bullets in NEWS.md.

**Key Decisions:**
- Model FIGARO ingestion as a parallel canonical importer rather than a polymorphic `import_suts()` branch, so composite-label parsing lives in one focused function.
- Strip the `CPA_` prefix at import so downstream code never has to distinguish raw vs. stripped product codes.
- Aggregate FIGARO final-demand columns into the canonical `FU_bas` variable at import, matching the WIOD long-table convention.
- Gate the replication test on `SUBE_WIOD_DIR`, not a bundled dataset, so CRAN and CI skip deterministically while researchers can verify numerically.
- Exclude `inst/extdata/wiod/` from the CRAN tarball via `.Rbuildignore`, so installed builds never pick up the local fallback and skip cleanly.
- Defer v1.1's named "Convenience" scope (one-call pipeline, batch helpers) to a future milestone rather than back-filling CNV-requirements.

**Issues Resolved:**
- FIGARO CSVs previously required manual pre-processing; `read_figaro()` now imports them directly.
- `.coerce_map()` silently fell through to positional matching on NACE-named mapping tables; this is now explicit synonym routing.
- No public path existed for reproducing paper numerics against packaged output; the gated replication test and vignette now make this a tested contract.
- `filter_paper_outliers()` was a dot-prefixed internal helper with implicit argument semantics; the exported signature documents `variables` and `apply_bounds`.

**Issues Deferred:**
- "Convenience" milestone scope (one-call pipeline, batch country/year helpers) — no CNV-requirements defined; carry forward to v1.2.
- Nyquist validation files (`*-VALIDATION.md` Nyquist schema) were not generated for phases 5 or 6. Optional retroactive `/gsd-validate-phase` runs can back-fill coverage; audit marked `nyquist.overall: not_enforced`.

**Technical Debt Incurred:**
- Pre-existing `tests/testthat/test-workflow.R:218` failure under `R CMD check --as-cran` (legacy-wrapper `Rscript` subprocess library-path isolation). `devtools::test()` passes 102/102; tarball `R CMD check --no-manual` exits Status: OK. Candidate for a v1.2 infrastructure phase.
- Under `devtools::load_all`, `devtools::test(filter='replication')` picks up the repo-local `inst/extdata/wiod/` fallback and surfaces a known ~4.4% methodological divergence that pre-dates this phase. Installed/CRAN builds skip cleanly. Optional follow-up: require explicit opt-in for the local fallback in `resolve_wiod_root()`.
