# Milestone v1.0: Package Workflow Hardening

**Status:** SHIPPED 2026-04-08
**Phases:** 1-4
**Total Plans:** 12

## Overview

This milestone turned `sube` from an already-functional brownfield R package into a tightly verified package-first workflow. The work hardened the core import-to-compute contracts, stabilized the Leontief comparison layer, aligned public documentation to the actual workflow, and finished with a clean release path plus a narrow migration bridge for script-era users.

## Milestone Summary

**Key Accomplishments:**
- Hardened the import, matrix, compute, and diagnostics workflow against malformed inputs and singular branches using shipped example data.
- Stabilized the comparison workflow from Leontief extraction through comparison tables, paper-style plots, and export semantics.
- Unified README, vignettes, pkgdown, and package help around a package-first workflow and discoverable sample-data contract.
- Hardened the release path with a broader GitHub Actions matrix, explicit test/build/check parity, and a clean tarball check.
- Preserved a working migration bridge for script-era users through the documented and tested legacy wrapper script.

**Key Decisions:**
- Treat `sube` as a brownfield package project, not a script collection.
- Use the package-first API as the canonical product surface.
- Keep legacy script support limited to a compatibility wrapper.
- Treat diagnostics and comparison/export semantics as explicit public contracts.
- Exclude `.planning/` from package builds so release artifacts stay package-facing.

**Issues Resolved:**
- Missing contractual coverage for malformed imports, mapping inputs, and compute diagnostics.
- Under-specified comparison output formats and export structures.
- Documentation drift between README, vignettes, pkgdown, and release notes.
- Mismatch risk between local maintainer commands and the GitHub Actions check path.
- Stale script-era contributor guidance in `AGENTS.md`.

**Issues Deferred:**
- None at milestone closeout.

**Technical Debt Incurred:**
- None retained after the final validation-artifact cleanup and passed milestone audit.
